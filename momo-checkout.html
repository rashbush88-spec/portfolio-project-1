Building MoMo-ready checkouts in Ghana

Abdul-Rashid Bushran · MarshalTech Hub · 6 min read

If you're building e-commerce in Ghana and you're only accepting card payments, you're leaving most of your customers behind. Mobile Money is the dominant payment method in Ghana — MTN MoMo alone has tens of millions of active users. This guide covers how to integrate it properly so your checkout doesn't frustrate people at the last step.

Why MoMo integration is different

Unlike card payments where the user enters details and you charge immediately, MoMo is asynchronous. You initiate a payment request, the user gets a prompt on their phone, they approve it, and then your server receives a callback. This means you need to handle:

  • An initial payment request to the MoMo API
  • A pending state on your UI while the user approves
  • A webhook/callback from the provider confirming success or failure
  • Timeout handling if the user never approves
The biggest mistake I see: Treating MoMo like a synchronous card charge. If you redirect the user immediately after initiating the request, you'll confirm orders that were never actually paid.

Choosing your provider

There are two main paths for Ghana:

  • Paystack — has built-in MoMo support for MTN, Vodafone Cash, and AirtelTigo. Great if you're already using Paystack for card payments. One API, multiple channels.
  • MTN MoMo API directly — more control, lower fees at scale, but more complex to set up and you only get MTN.

For most SMB projects I build, Paystack is the right call. The unified API means you handle one integration and get cards + all MoMo networks.

Basic flow with Paystack

Here's the simplified flow for a MoMo payment via Paystack: 

// 1. Initiate payment on your backend (Node.js example)
const response = await fetch('https://api.paystack.co/transaction/initialize', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${process.env.PAYSTACK_SECRET_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: customer.email,
    amount: orderTotal * 100,  // Paystack uses pesewas
    currency: 'GHS',
    channels: ['mobile_money'],
    mobile_money: {
      phone: customer.phone,
      provider: 'mtn'  // or 'vodafone', 'airtel'
    }
  })
});

const { data } = await response.json();
// data.authorization_url — redirect user here
// data.reference — save this to match the webhook

Handling the callback

Set up a webhook endpoint that Paystack will call once the payment is confirmed. Always verify the webhook signature — never trust the payload without validating it came from Paystack. 

// Webhook endpoint (Express.js)
app.post('/webhooks/paystack', express.raw({type: 'application/json'}), (req, res) => {
  const hash = crypto
    .createHmac('sha512', process.env.PAYSTACK_SECRET_KEY)
    .update(req.body)
    .digest('hex');

  if (hash !== req.headers['x-paystack-signature']) {
    return res.status(401).send('Invalid signature');
  }

  const event = JSON.parse(req.body);

  if (event.event === 'charge.success') {
    const ref = event.data.reference;
    // Mark order as paid in your database
    await markOrderPaid(ref);
  }

  res.sendStatus(200);
});

UX tips from real deployments

After building several MoMo-integrated platforms in Ghana, here's what actually matters for the user experience:

  • Tell users what to expect. After initiating, show a screen that says "Check your phone — approve the MTN MoMo prompt to complete your order." Most users haven't done this before.
  • Use a polling mechanism on the frontend so the page auto-updates when payment is confirmed. Don't make users manually refresh.
  • Set a timeout of 3–5 minutes. If no confirmation arrives, expire the pending order and let the user retry.
  • Log everything. MoMo transactions can fail silently. A thorough log of every request, response, and callback saves hours of debugging.
On one project, we discovered that about 12% of users initiated a MoMo payment but didn't approve the prompt on their phone. A simple follow-up SMS ("Your order is waiting — approve the MoMo request to confirm") recovered about half of those.

Testing without a live account

Paystack's test mode supports MoMo simulation. Use the test phone number 0551234987 with any MoMo provider — it will trigger a simulated approval. This lets you test the full async flow without a live MTN account.

Building payment systems is one of the most impactful things you can do for Ghanaian businesses. Get this right and you remove the biggest drop-off point in your checkout.
